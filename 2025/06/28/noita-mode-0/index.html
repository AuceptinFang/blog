新建文件夹

第一步，在mods文件夹下新建文件夹

一个模组至少包括两个文件，mod.xml和init.lua

关于.xml文件

在pom.xml里已经被折磨过见过了，重新从头认识一下：eXtensible Markup Language，意为”可扩展标记语言”。

声明（可以没有，noita的文件都没有）：

<?xml version="1.0" encoding="UTF-8"?>

交代编码方式

元素和属性

元素用成对的标签包裹内容，属性附着在起始标签上提供附加信息，必须加引号

<Mod
	name="Daily Practice Run"
	description="You can practice by replaying the same seed within 24 hours."
	ui_newgame_name="$gamemode_daily_practice_run"
	ui_newgame_description="$gamemode_daily_practice_run_desc"
	ui_newgame_gfx_banner_bg="mods/daily_practice/files/menu_banner_background.png"
	ui_newgame_gfx_banner_fg="mods/daily_practice/files/menu_banner_overlay.png"
	request_no_api_restrictions="0"

	> 
</Mod>

空元素

自闭合

<br />
<item type="wand" />

注释

同html

<!-- 这是一个魔法物品列表 -->

实体引用

一种用特殊字符来表示另一些字符的机制。比如”<”会被解析成一个标签开头，可以用一套以&开头;结尾的字符替代

字符 实体引用 含义
< < less than（小于）
> > greater than（大于）
& & ampersand（&符号）
“ " quotation mark（双引号）
‘ ' apostrophe（单引号）

CDATA 区块

当你不想让某些字符被解析成标签，可以放入 CDATA：

<script><![CDATA[
  if (hp < 10) then explode()
]]></script>

ESC系统——游戏如何运行

要了解Noita的游戏逻辑，就需要了解“实体组件系统”（ECS，Entity Component System）
noita以 Lua 和 XML 语言实现ESC，游戏中每个独立的 “东西”（敌人、子弹、魔杖、物品、包括玩家）都是实体（Entity），只不过构成它们的组件不同。

Base file标签是一个继承，后续是添加更多新组件，比如动物AI组件等等

继承规则详解

  • include_children="1": 继承所有子实体
  • 同名组件会被完全覆盖，而不是合并
  • 新组件会被添加到现有组件列表
  • 支持多级继承链
  • 子实体也支持继承机制

虚拟文件系统（VFS）——模组如何生效

VFS工作原理:

  1. 文件路径重定向: 当游戏请求读取某个文件时，VFS首先检查是否有模组提供了该文件的替换版本
  2. 内容合并: 对于支持追加的文件（如Lua脚本），VFS将原始内容与模组内容合并
  3. 优先级处理: 多个模组修改同一文件时，按照模组加载顺序确定优先级
  4. 透明访问: 游戏代码无需修改，通过VFS透明地访问合并后的内容
    物理文件系统布局:
Noita/
├── data/                    # 游戏原始数据
│   ├── scripts/
│   ├── entities/
│   └── materials.xml
├── mods/
│   ├── mod_a/
│   │   ├── mod.xml
│   │   ├── init.lua
│   │   └── files/
│   │       ├── scripts/
│   │       └── entities/
│   └── mod_b/
│       └── files/

虚拟文件系统映射:
data/scripts/gun/gun_actions.lua -> [原始文件] + [mod_a追加] + [mod_b追加]
data/entities/animals/sheep.xml  -> [mod_a覆盖版本] (如果存在)
data/materials.xml              -> [原始文件] + [mod_a材料] + [mod_b材料]
    也就是说，VFS系统给我们提供了方便地修改/增添游戏内容的机制

发现决定了变形目标的代码是: polymorph_target=”[RANDOM]”
作为对比，这是变形魔药（变羊）对应的部分：polymorph_target=”data/entities/animals/sheep.xml”

那么模组的制作思路就一目了然了——把蠕虫文件换进去。而新的问题也出现了，蠕虫有五种：大、中、小、地狱的、艺之神殿的，而xml如果写路径只能放一个

只好进一步研究[RANDOM]的实现了，有一个polymorph表，游戏启动时会把可变形的生物加进去

需要借助nolla提供的api来修改，这里要用到PolymorphTableSet和PolymorphTableGet

实现代码：

<!-->mod.xml<!-->
<Mod
        name="Always Worm"
        description="将混乱变形效果限制为只在基础蠕虫系列中随机变形 (worm, worm_big, worm_tiny, worm_end, worm_skull)"
        author="nil"
        request_no_api_restrictions="0"
        is_game_mode="0"
>
</Mod>
-- init.lua
-- 设置polymorph表只包含各种蠕虫
function set_worm_only_polymorph()
	local worm_entities = {
        "data/entities/animals/worm.xml",           -- 普通蠕虫
		"data/entities/animals/worm_big.xml",       -- 大蠕虫
		"data/entities/animals/worm_tiny.xml",      -- 小蠕虫
		"data/entities/animals/worm_end.xml",       -- 地狱蠕虫
		"data/entities/animals/worm_skull.xml"      -- 幽灵蠕虫
	}
	
	PolymorphTableSet(worm_entities, false)     -- 核心函数
	PolymorphTableSet({}, true)
end

-- 模组初始化
function OnModPostInit()
	set_worm_only_polymorph()
end

-- 世界初始化时也设置一次
function OnWorldInitialized()
	set_worm_only_polymorph()
end

-- nothing
function OnWorldPreUpdate()
	
end
