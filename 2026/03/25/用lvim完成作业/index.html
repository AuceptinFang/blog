用lvim迈向终端全栈

发表于2026-03-25|更新于2026-03-26|家要有家的样子
|浏览量:

本系列的上一篇用wezterm、fish和tmux大大改善了终端的使用体验，随着最近对vim的操作变得熟练，以前依旧要依赖GUI的IDE似乎也可以搬进终端里了

故事的起因是有些作业相比于手写，还是打字更舒服，但是学校的作业平台又不能直接提交markdown，于是……

md -> pdf

在~/.config/lvim里增加插件：

lvim.plugins = {
  {
    "iamcco/markdown-preview.nvim",
    cmd = { "MarkdownPreviewToggle", "MarkdownPreview", "MarkdownPreviewStop" },
    build = "cd app && npm install",
    init = function()
      vim.g.mkdp_filetypes = { "markdown" }
    end,
    ft = { "markdown" },
  }
}

再增加一个快捷键：

lvim.builtin.which_key.mappings["m"] = {
  name = "Markdown",
  m = { "<cmd>MarkdownPreviewToggle<cr>", "Toggle Preview" },
}

于是可以 m + m在浏览器里打开预览，同时可以用浏览器自带的打印功能保存为pdf

更多lvim操作

space + e 打开目录 a新建文件，可以方便地编辑多个文件

ctrl + \打开浮窗终端，默认应该是insert模式，如果不是按i进入。这样写完代码运行，一般情况下就不用单开一页了

gd go to definition，跳转到函数定义，可以跨文件操作，K 查看声明

ctrl w + v竖直分屏，ctrl + 左右方向移动窗口

还有一揽子vim快捷键

ctrl d下翻半屏

e下一个单词末尾，b上一个单词开头，w下一个单词开头

V进入View模式，然后选中多行<或>调整缩进

.在当前位置应用上一次操作

*跳转到选中词出现的其他位置

cw改词（删除当前指向的词并进入insert模式）ciw change inner word 不需要让光标置于当前词的开头
r替换

/name 文件内名称搜索

wezterm动态改透明度

有的时候截屏终端，透明的背景会泄露不必要的信息，不透明的又不好看，这就体现出lua做配置文件的灵活之处了：

-- .wezterm.lua
table.insert(config.keys, {
  key = '-',
  mods = 'ALT',
  action = wezterm.action_callback(function(window)
    local overrides = window:get_config_overrides() or {}
    local current = overrides.window_background_opacity or config.window_background_opacity or 1.0
    overrides.window_background_opacity = math.max(0.3, current - 0.03)
    window:set_config_overrides(overrides)
  end),
})

table.insert(config.keys, {
  key = '=',
  mods = 'ALT',
  action = wezterm.action_callback(function(window, pane)
          local overrides = window:get_config_overrides() or {}
          local current = overrides.window_background_opacity or config.window_background_opacity or 1.0
          overrides.window_background_opacity = current + 0.03
    window:set_config_overrides(overrides)
  end),
})
